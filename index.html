云计算技术作业二

2019级数据班

李文婷 201918130233

2-1、列举威胁作用者有哪些,并简要说明。

  • 匿名攻击者：是云中没有权限的、不被信任的云服务用户。通常是一个外部软件程序，通过公网（云边界的外部）发动网络攻击。

  1. 恶意服务作用者：能截取并转发云内的网络流量，试图恶意地使用和篡改数据。它通常是带有被损害的或恶意逻辑的服务代理（或伪装成服务代理的程序）。也有可能是能够远程截取并破坏消息内容的外部程序。

  2. 授信的攻击者：又称恶意租户，是经过授权的云服务用户，具有合法的证书。通常通过滥用合法的证书或通过挪用敏感和保密的信息，在云的信任边界内部发动攻击。

  3. 恶意的内部人员：是人为的威胁作用者。通常是现任或前任雇员。拥有访问云用户IT资源的管理特权，会带来极大的破坏可能性。

2-2、列举云安全威胁有哪些,并简要说明。

  • 流量窃听：当数据在传输到云中或在云内部传输时（通常是从云用户到云提供者）被恶意的服务作用者被动地截获，用于非法的信息收集之目的。通常是由截取网络流量的恶意服务作用者实施的。这种攻击的目的是直接破坏数据的保密性。

  1. 恶意媒介：消息被恶意服务作用者截获并且被篡改，可能会破坏消息的完整性和保密性。很有可能在把消息转发到目的地之前插入有害的数据。通常是由截取网络流量的恶意服务作用者实施的。

  2. 拒绝服务：使IT资源过载（因为请求过多）至无法正确运行。这些请求意在使IT资源性能限于瘫痪或者不可用。

  3. 授权不足：错误地授予了攻击者访问权限或者是授权太宽泛，导致攻击者能够访问本应该受到保护的IT资源。其一种变种是弱认证（用户使用了弱密码）。

  4. 虚拟化攻击：利用虚拟化环境中的漏洞，获得了对底层物理硬件未被授权的访问，来危害虚拟化平台的保密性、完整性和可用性。

  5. 重叠的信任边界：攻击者可以利用多个云用户共享的、基于云的IT资源，攻击底层的物理服务器及其他用户使用的虚拟服务器。

云计算技术作业一

2019级数据班

李文婷 201918130233

1-1、云使能技术主要包括哪些技术组件？简要说明其关键特征。

答：云使能技术主要包括宽带网络和Internet架构、数据中心技术、虚拟化技术、Web技术、多租户技术和服务技术。各技术组件的关键特征分别如下：

  1. 宽带网络和Internet架构：云用户和云提供者通常利用Internet进行通信，Internet以无中心的供给和管理模型为基础，不受任何集中式实体的控制。

  2. 数据中心技术：数据中心包含了物理和虚拟的IT资源，以标准化商用硬件为基础，用模块化架构进行设计，整合多个相同的基础设施模块和设备，具备可扩展性、可增长性和快速更换硬件的特点。

  3. 虚拟化技术：

    • 在虚拟服务器上运行的客户操作系统和应用软件，都不会感知到虚拟化的过程。
    • 硬件无关性：在一个IT硬件平台上配置操作系统和安装应用软件会导致许多软硬件依赖关系，非虚拟环境下硬件的变化，将导致操作系统的重新配置。
    • 隔离：在同一服务器上的虚拟机之间互相隔离，改变了之前单台物理机服务器只能挂单个应用的格局。
    • 服务器整合（分区）：虚拟化软件提供的协调功能，可以在一个虚拟主机上同时创建多个虚拟服务器。虚拟化技术允许不同虚拟服务器共享一个物理服务器，这就是服务器整合(server consolidation)。

  4. Web技术：常被作为云服务的实现介质，并在前端作为管理接口，用于远程管理云IT资源。其两个基本组件是Web浏览器客户端和Web服务器。

  5. 多租户技术：多租户应用的一般特点：

    • 使用隔离(usage isolation)：一个租户的使用行为不会影响到该应用对其他租户可用性和性能。
    • 数据安全(data security)：租户不能访问其他租户的数据。
    • 可恢复性(recovery)：每个租户的数据备份和恢复过程都是分别执行的。
    • 应用升级(application upgrade)：共享软件构件的同步升级不会对租户造成负面影响。
    • 可扩展性(scalability)：根据现有租户增长的使用需求或租户数量增加来扩展应用。
    • 使用计费(metered usage)：根据租户实际使用的应用处理和功能来收费。
    • 数据层隔离(data tier isolation)：租户拥有独立的且与其他租户隔离的数据库、表格和模式。

  6. 服务技术：是云计算的基石，它形成了“作为服务”的云交付模型的基础。主要包括：基于Web的服务（如Web服务和REST服务）、服务代理和服务中间件。

1-2、列举至少3种虚拟化软件，并概括其特点。

  • VMware Workstation：
    • 使用Unity来集成客户机与宿主机.
    • 更加强大的VM录制与回放功能.
    • 支持智能卡和相关读卡器.
    • 增强型ACE.
    • 改进型3D图形支持.

  1. Oracle VM VirtualBox

    • 支持64位客户端操作系统，即使主机使用32位CPU
    • 支持SATA硬盘NCQ技术
    • 虚拟硬盘快照
    • 无缝视窗模式（须安装客户端驱动）
    • 能够在主机端与客户端共享剪贴簿（须安装客户端驱动）
    • 在主机端与客户端间建立分享文件夹（须安装客户端驱动）
    • 内建远端桌面服务器，实现单机多用户　
    • 支持VMware VMDK磁盘档及Virtual PC VHD磁盘档格式
    • 支持VT-x与AMD-V硬件虚拟化技术
    • iSCSI支持
    • USB与USB2.0支持

  2. Citrix XenDesktop：

    • 可以根据USB设备的VID, PID, REL, Class, SubClass, Prot tags等描述字段进行识别并访问控制；
    • 可以设置剪贴板、文件复制、USB设备等的单向传输，这点对于研发环境尤为重要，配置为只进不出；
    • 可以根据时间、终端位置和设备类型控制访问权限和虚拟机功能。
